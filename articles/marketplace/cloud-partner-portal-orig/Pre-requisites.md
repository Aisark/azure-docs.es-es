# <a name="prerequisites-for-application-curation"></a>Requisitos previos para la protección de aplicaciones

Microsoft requiere revisiones específicas con el fin de validar que se cumplen los requisitos siguientes:

- El código personalizado de un asociado cumple las directrices de Microsoft.
- Se puede implementar correctamente un paquete de soluciones de Microsoft Dynamics Lifecycle Services (LCS).
- Se pueden completar las transacciones.

Actualmente, los asociados deben demostrar que se cumplen estos requisitos realizando implementaciones de prueba y, a continuación, compartiendo los resultados con Microsoft. No se implementará ningún código en un entorno de cliente que Microsoft no haya validado. Los asociados deben completar los siguientes artefactos y las siguientes pruebas de protección:

- Informe de análisis de código (CAR)
- Modelador de procesos empresariales (BPM) o scripts de prueba
- Nombre para mostrar y descripción
- Copia de seguridad de bases de datos empresariales
- Paquetes de datos y paquetes de datos de proceso (PDP)
- Metodología
- Archivos binarios (opcionales)
- Paquetes que se pueden implementar
- Modelos (código y pruebas)
- Contenido de marketing

Para obtener más información sobre los requisitos técnicos y de procesos, consulte [validar aplicaciones de finanzas y operaciones](https://docs.microsoft.com/dynamics365/operations/dev-itpro/lcs-solutions/app-validation-lcs-solutions).  Además, si aún no lo ha hecho, revise el [Office 365, Dynamics 365, PowerApps y Power BI ofrecen guía para la publicación](../appsource-offer-publishing-guide.md).
