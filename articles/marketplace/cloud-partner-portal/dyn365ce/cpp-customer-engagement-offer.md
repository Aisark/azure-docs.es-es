# <a name="dynamics-365-for-customer-engagement-offer"></a>Oferta de Dynamics 365 for Customer Engagement

|     |     |
| --- | --- |
| <div style="font-size:1em"> En esta sección se explica cómo publicar una solución de Dynamics 365 for Customer Engagement en Microsoft [AppSource Marketplace](https://appsource.microsoft.com). Todas las aplicaciones de Dynamics 365 for Customer Engagement (ventas, servicio, servicio de proyectos y servicio de campo) deben someterse a nuestro proceso de certificación y admitir una experiencia de evaluación. El proceso de certificación comprueba su solución en busca de requisitos estándar, compatibilidad y procedimientos adecuados. La experiencia de evaluación permite a los usuarios implementar la solución en un entorno de Dynamics 365 en vivo. </div>|  ![Icono de Dynamics 365 for Customer Engagement](./media/dynce-icon1.png)  |


## <a name="publishing-overview"></a>Introducción a la publicación

En el siguiente vídeo, [Deliver a compelling experience on Microsoft AppSource (Build 2018)](https://www.youtube.com/watch?v=WWBFuNHC0J4) [Entrega de una experiencia atractiva en Microsoft AppSource (Build 2018)], se ofrece una introducción a AppSource Marketplace y se presentan las ventajas de este ecosistema. Luego se presenta un ejemplo de una aplicación personalizada de Dynamics 365 for Customer Engagement, se indican las ventajas de proporcionar una evaluación de versión de prueba y se explica cómo publicar en AppSource.

> [!Video https://channel9.msdn.com/events/Build/2018/THR1202/player]


## <a name="publishing-process"></a>Proceso de publicación

Los artículos siguientes de esta sección le guiarán por el proceso de creación y publicación de una aplicación de Dynamics 365 for Customer Engagement.
