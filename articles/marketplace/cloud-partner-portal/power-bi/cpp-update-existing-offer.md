# <a name="update-a-power-bi-app-offer"></a>Actualización de una oferta de aplicación de Power BI

>[!Important]
>A partir del 13 de abril de 2020, se comenzará a trasladar la administración de las ofertas de aplicaciones de Power BI al Centro de partners. Después de la migración, las ofertas se crearán y administrarán en el Centro de partners. Siga las instrucciones que se indican en [Información general sobre la creación de aplicaciones de Power BI](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-power-bi-app-offer) para administrar las ofertas migradas.

En este artículo se muestra cómo actualizar la oferta de aplicación de Microsoft Power BI en [Cloud Partner Portal](https://cloudpartner.azure.com/) y, luego, volver a publicar la oferta. Estas son algunas de las razones comunes para actualizar una oferta:

- Para actualizar el contenido de la aplicación en Power BI y obtener una dirección URL de instalación de la aplicación recién empaquetada.
- Para actualizar los metadatos de Azure Marketplace para la oferta (información y recursos de ventas, marketing o soporte técnico).
 
Use las características **Comparar** e **Historial** del portal para hacer un seguimiento de los cambios como estos.

## <a name="common-update-operations"></a>Operaciones de actualización comunes

Algunos atributos de una oferta de aplicación de Power BI no pueden cambiarse una vez publicada la oferta en AppSource. Entre los cambios no permitidos se incluyen el **Id. de oferta** y el **Id. de publicador**. Pero aún puede cambiar una amplia gama de características. Estos son algunos cambios habituales.

### <a name="update-app-content-in-power-bi"></a>Actualizar el contenido de la aplicación en Power BI

A menudo, las aplicaciones de Power BI se actualizan periódicamente con nuevo contenido, revisiones de seguridad, características, etc. Cuando se cambia una aplicación, actualice su dirección URL de instalación siguiendo estos pasos:

1.  Inicie sesión en [Cloud Partner Portal](https://cloudpartner.azure.com/).
2.  En **Todas las ofertas**, busque la oferta que quiere actualizar.
3.  En la pestaña **Información técnica**, escriba una nueva dirección URL de instalación.
4.  Haga clic en **Publicar** para iniciar el flujo de trabajo para publicar la nueva versión de la aplicación en AppSource.


### <a name="update-the-offers-marketplace-metadata"></a>Actualizar los metadatos de Marketplace para una oferta

Los metadatos de Marketplace para la oferta incluyen el nombre de su empresa, los logotipos y otra información. Para actualizar los metadatos, siga estos pasos:

1.  Inicie sesión en [Cloud Partner Portal](https://cloudpartner.azure.com/).
2.  En **Todas las ofertas**, busque la oferta que quiere actualizar.
3.  En la pestaña**Detalles del escaparate electrónico**, cambie los metadatos según las instrucciones del artículo [Pestaña de detalles del escaparate electrónico de aplicación de Power BI](./cpp-storefront-details-tab.md).
4.  Seleccione **Publicar** para iniciar el flujo de trabajo para publicar los cambios.


## <a name="the-compare-feature"></a>La característica Comparar

Al actualizar una oferta publicada, puede comprobar los cambios hechos. Para usar la característica **Comparar**:

1.  En cualquier punto del proceso de edición, seleccione el botón **Comparar**.

    ![Botón Comparar](./media/compare-feature-button.png)

2.  Vea las versiones en paralelo de los recursos de marketing y metadatos.


## <a name="history-of-publishing-actions"></a>Historial de acciones de publicación

Para ver el historial de publicación de la oferta, en el lado izquierdo de Cloud Partner Portal, abra la pestaña **Historial**. Aquí podrá ver un historial de acciones con marca de tiempo en sus ofertas de AppSource.
