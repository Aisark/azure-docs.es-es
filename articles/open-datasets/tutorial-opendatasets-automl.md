# <a name="tutorial-build-a-regression-model-with-automated-machine-learning-and-open-datasets"></a>Tutorial: Creación de un modelo de regresión con aprendizaje automático automatizado y Open Datasets

En este tutorial, sacará el máximo provecho de la comodidad de Azure Open Datasets (y de la eficacia de Azure Machine Learning Service) para crear un modelo de regresión que prediga las tarifas de taxi de Nueva York. Descargue fácilmente datos disponibles de forma pública sobre taxis, días festivos y condiciones meteorológicas y, después, configure un experimento de aprendizaje automático automatizado mediante Azure Machine Learning Service. Este proceso acepta valores de configuración y datos de entrenamiento y, después, itera automáticamente las combinaciones de diferentes métodos de estandarización o normalización de características, modelos y configuraciones de hiperparámetros para llegar al mejor modelo.

En este tutorial, aprenderá a realizar las tareas siguientes:

- Configurar un área de trabajo de Azure Machine Learning
- Configurar un entorno de Python local
- Acceder, transformar y combinar datos mediante Azure Open Datasets
- Entrenar un modelo de regresión con aprendizaje automático automatizado
- Calcular la precisión del modelo

## <a name="prerequisites"></a>Requisitos previos

Este tutorial cuenta con los siguientes requisitos previos.

* Un área de trabajo de Azure Machine Learning Service.
* Un entorno de Python 3.6

### <a name="create-a-workspace"></a>Crear un área de trabajo

Siga las [instrucciones](https://docs.microsoft.com/azure/machine-learning/service/setup-create-workspace#portal) para crear un área de trabajo a través de Azure Portal, si aún no la tiene. Después de crearla, tome nota del nombre del área de trabajo, el nombre del grupo de recursos y el identificador de suscripción.

### <a name="create-a-python-environment"></a>Crear un entorno de Python

En este ejemplo se usa un entorno de Anaconda con cuadernos de Jupyter, pero puede ejecutar este código en cualquier entorno 3.6.x y usar cualquier editor de texto o IDE. Utilice los pasos siguientes para crear un nuevo entorno de desarrollo.

1. Si aún no lo tiene, [descargue](https://www.anaconda.com/distribution/) e instale Anaconda y elija la **versión 3.7 de Python**.
1. Abra un símbolo del sistema de Anaconda y cree un nuevo entorno. La creación del entorno tardará unos minutos mientras se descargan los componentes y los paquetes.
    ```
    conda create -n tutorialenv python=3.6.5
    ```
1. Active el entorno.
    ```
    conda activate tutorialenv
    ```
1. Habilite los kernels de IPython específicos del entorno.
    ```
    conda install notebook ipykernel
    ```
1. Cree el kernel.
    ```
    ipython kernel install --user
    ```
1. Instale los paquetes que necesita para este tutorial. Estos paquetes son grandes y tardan entre 5 y 10 minutos en instalarse.
    ```
    pip install azureml-sdk[automl] azureml-opendatasets
    ```
1. Inicie un kernel de cuaderno desde su entorno.
    ```
    jupyter notebook
    ```

Después de completar estos pasos, clone el [repositorio del cuaderno de Open Datasets](https://github.com/Azure/OpenDatasetsNotebooks) y abra el cuaderno **tutorials/taxi-automl/01-tutorial-opendatasets-automl.ipynb** para ejecutarlo.

## <a name="download-and-prepare-data"></a>Descargar y preparar los datos

Importe los paquetes necesarios. El paquete de Open Datasets contiene una clase que representa cada origen de datos (por ejemplo, `NycTlcGreen`) para filtrar fácilmente los parámetros de fecha antes de la descarga.


```python
from azureml.opendatasets import NycTlcGreen
import pandas as pd
from datetime import datetime
from dateutil.relativedelta import relativedelta
```

Empiece por crear una trama de datos para almacenar los datos de taxis. Cuando se trabaja en un entorno que no es de Spark, Open Datasets solo permite descargar un mes de datos a la vez con determinadas clases, para así evitar que se presenten errores `MemoryError` con grandes conjuntos de datos. Para descargar un año de datos de taxi, recupere de forma iterativa un mes a la vez y, antes de agregarlo a `green_taxi_df`, realice un muestreo aleatorio de 2000 registros de cada mes para evitar el sobredimensionamiento de la trama de datos. Después, obtenga una vista previa de los datos.

>[!NOTE]
> Open Datasets incluye clases para creación de reflejos de la base de datos que permiten trabajar en entornos de Spark en los que el tamaño de los datos y la memoria no suponen ningún problema.

```python
green_taxi_df = pd.DataFrame([])
start = datetime.strptime("1/1/2016", "%m/%d/%Y")
end = datetime.strptime("1/31/2016", "%m/%d/%Y")

for sample_month in range(12):
    temp_df_green = NycTlcGreen(start + relativedelta(months=sample_month), end + relativedelta(months=sample_month)) \
        .to_pandas_dataframe()
    green_taxi_df = green_taxi_df.append(temp_df_green.sample(2000))

green_taxi_df.head(10)
```

Ahora que se han cargado los datos iniciales, defina una función para crear diversas características basadas en tiempo a partir del campo de fecha y hora de recogida. Así, se crearán nuevos campos para el número de mes, día del mes, día de la semana y hora del día. Además, el modelo podrá descomponerse en factores según la estacionalidad basada en tiempo. La función también agrega una característica estática para el código de país, para combinar los datos de días festivos. Use la función `apply()` en la trama de datos para aplicar de forma iterativa la función `build_time_features()` a cada fila de los datos de taxis.


```python
def build_time_features(vector):
    pickup_datetime = vector[0]
    month_num = pickup_datetime.month
    day_of_month = pickup_datetime.day
    day_of_week = pickup_datetime.weekday()
    hour_of_day = pickup_datetime.hour
    country_code = "US"

    return pd.Series((month_num, day_of_month, day_of_week, hour_of_day, country_code))


green_taxi_df[["month_num", "day_of_month", "day_of_week", "hour_of_day", "country_code"]
              ] = green_taxi_df[["lpepPickupDatetime"]].apply(build_time_features, axis=1)
green_taxi_df.head(10)
```

## <a name="train-a-model"></a>Entrenamiento de un modelo

Ahora, use los datos preparados para entrenar un modelo de Machine Learning automatizado. Comience por dividir `final_df` en características (valores de X) y etiquetas (valores de Y), que para este modelo son la tarifa de taxi.

```python
y_df = final_df.pop("totalAmount")
x_df = final_df
```

Ahora puede dividir los datos en conjuntos de entrenamiento y prueba mediante la función `train_test_split()` de la biblioteca `scikit-learn`. El parámetro `test_size` determina el porcentaje de datos que se va a asignar a las pruebas. El parámetro `random_state` establece un valor de inicialización para el generador de números aleatorios, de forma que las divisiones de entrenamiento o prueba son deterministas.


```python
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    x_df, y_df, test_size=0.2, random_state=222)
```

### <a name="load-workspace-and-configure-experiment"></a>Cargar el área de trabajo y configurar el experimento

Cargue el área de trabajo de Azure Machine Learning Service mediante la función `get()` con su información de suscripción y área de trabajo. Cree un experimento en el área de trabajo para almacenar y supervisar las ejecuciones del modelo.


```python
from azureml.core.workspace import Workspace
from azureml.core.experiment import Experiment

workspace = Workspace.get(subscription_id="<your-subscription-id>",
                          name="<your-workspace-name>", resource_group="<your-resource-group>")
experiment = Experiment(workspace, "opendatasets-ml")
```

Cree un objeto de configuración para el experimento mediante la clase `AutoMLConfig`. Adjunte sus datos de entrenamiento y, de forma adicional, especifique las configuraciones y parámetros que controlan el proceso de entrenamiento. Los parámetros tienen las finalidades siguientes:

* `task`: tipo de experimento que se va a ejecutar.
* `X`: características de entrenamiento.
* `y`: etiquetas de entrenamiento.
* `iterations`: número de iteraciones que se va a ejecutar. Cada iteración prueba combinaciones de diferentes métodos de estandarización/normalización de características y diferentes modelos mediante varias opciones de configuración de hiperparámetros.
* `primary_metric`: métrica principal que se va a optimizar durante el entrenamiento del modelo. El modelo de ajuste más adecuado se elegirá según esta métrica.
* `preprocess`: controla si el experimento puede preprocesar los datos de entrada (al administrar los datos que faltan, convertir texto a numérico, etc.).
* `n_cross_validations`: Número de divisiones de la validación cruzada que se realizarán cuando no se especifiquen datos de validación.


```python
from azureml.train.automl import AutoMLConfig

automl_config = AutoMLConfig(task="regression",
                             X=X_train.values,
                             y=y_train.values.flatten(),
                             iterations=20,
                             primary_metric="spearman_correlation",
                             preprocess=True,
                             n_cross_validations=5
```


## <a name="test-model-accuracy"></a>Probar la precisión del modelo

```python
y_predict = fitted_model.predict(X_test.values)
```

Calcule el error cuadrático medio de los resultados. Use la trama de datos `y_test` y conviértala en una lista `y_actual` para compararla con los valores predichos. La función `mean_squared_error` toma dos matrices de valores y calcula el error medio al cuadrado entre ellos. Al tomar la raíz cuadrada del resultado, se produce un error en las mismas unidades que la variable "y": costo. Indica, aproximadamente, cuánto se alejan las predicciones de la tarifa de taxi de las reales, a la vez que pondera en gran medida los errores grandes.


```python
from sklearn.metrics import mean_squared_error

y_actual = y_test.values.flatten().tolist()
```

Ejecute el siguiente código para calcular el error medio absoluto porcentual (MAPE) usando los conjuntos de datos `y_actual` y `y_predict` completos. Esta métrica calcula una diferencia absoluta entre cada valor predicho y real, y suma todas las diferencias. A continuación, expresa esa suma como porcentaje del total de los valores reales.


```python
sum_actuals = sum_errors = 0

for actual_val, predict_val in zip(y_actual, y_predict):
    abs_error = actual_val - predict_val
    if abs_error < 0:
        abs_error = abs_error * -1

    sum_errors = sum_errors + abs_error
    sum_actuals = sum_actuals + actual_val

mean_abs_percent_error = sum_errors / sum_actuals
print("Model MAPE:")
print(mean_abs_percent_error)
print()
print("Model Accuracy:")
print(1 - mean_abs_percent_error)
```
