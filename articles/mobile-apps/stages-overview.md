# <a name="different-stages-in-mobile-application-development"></a>Diferentes fases en el desarrollo de aplicaciones móviles
La compilación de una aplicación móvil implica varias fases y los desarrolladores pueden aprovechar fácilmente las ventajas de los servicios, las herramientas y las tecnologías de Microsoft para crear aplicaciones móviles de alta calidad.

## <a name="app-development"></a>Desarrollo de aplicaciones
Los desarrolladores pueden compilar aplicaciones cliente de plataforma única nativa con lenguajes como Objective-C y Java, aplicaciones multiplataforma con Xamarin, React Native, Unity e híbridos con Cordova. Pueden hacer uso de los IDE y los editores de código, como Visual Studio, Visual Studio para Mac o Visual Studio Code en equipos PC y Mac, para construir sus aplicaciones cliente.

## <a name="plan-and-design-with-back-end-services"></a>Planeamiento y diseño con servicios de back-end
 Microsoft ofrece diversos servicios para configurar un back-end seguro, escalable y confiable para su aplicación móvil. Los desarrolladores pueden crear interfaces API y desencadenadores sin servidor sin tener que mantener sus propios servidores. Pueden aprovechar servicios tales como las notificaciones push, la sincronización de datos sin conexión, la autenticación y los datos, que juntos forman un back-end móvil completo y amplio como servicio que admite diversos escenarios de construcción de aplicaciones. La creación de un servicio back-end tiene muchos requisitos:
   - Limitar el almacenamiento de datos en el dispositivo.
   - Sincroniza datos en varios dispositivos.
   - Asegurarse de que la aplicación se ejecuta sin conectividad de red.
   - Enviar notificaciones a los usuarios.
   - Reducir el consumo de batería.
   - Compilar un back-end escalable, confiable y seguro.
   - Mantenimiento automático del servidor.

## <a name="devops-and-continuous-monitoring"></a>DevOps y supervisión continua
Los desarrolladores que crean aplicaciones para dispositivos móviles y centradas en la nube pueden aprovechar los servicios de integración continua (CI), entrega continua (CD) y supervisión continua para producir compilaciones nativas para todas las plataformas, ejecutar pruebas de automatización de la interfaz de usuario en miles de dispositivos físicos, automatizar sus versiones a través de canales de pruebas beta o implementarlas directamente en tiendas de aplicaciones. Además, pueden recopilar análisis de uso, información de bloqueo y excepciones sobre sus aplicaciones.

## <a name="additional-services"></a>Servicios adicionales
Hay muchos servicios de Azure que pueden consumirse en función del tipo y la naturaleza de la aplicación que se está construyendo.
  - Azure Search para buscar todos los tipos de datos internos, con consultas rápidas.
  - Azure Cognitive Services para usar inteligencia artificial en la aplicación.
  - Azure SignalR para habilitar la comunicación en tiempo real en la aplicación.
