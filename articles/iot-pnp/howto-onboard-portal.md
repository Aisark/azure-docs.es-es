# <a name="onboard-to-the-azure-certified-for-iot-portal"></a>Incorporación al portal de Azure Certified for IoT

Puede usar el [portal de Azure Certified for IoT](https://aka.ms/ACFI) para:

- Completar el proceso de certificación del dispositivo IoT.
- Buscar modelos de funcionalidad del dispositivo IoT Plug and Play (versión preliminar). Estos modelos se pueden usar para crear rápidamente dispositivos listos para IoT e integrarlos con soluciones.
- Publicar un modelo de funcionalidad del dispositivo en el repositorio público.

## <a name="sign-in"></a>Iniciar sesión

Use su _cuenta profesional_ de Microsoft para iniciar sesión en el [portal de Azure Certified for IoT](https://aka.ms/ACFI). El portal comprueba la pertenencia al Centro de partners de Microsoft cuando inicia sesión:

- Si su organización ya es miembro del [Centro de partners de Microsoft](https://docs.microsoft.com/partner-center/), se le concede acceso a todas las funciones del portal.
- Si su organización no es miembro del [Centro de partners de Microsoft](https://docs.microsoft.com/partner-center/), puede ejecutar pruebas de certificación y examinar los repositorios del modelo de funcionalidad del dispositivo. No se puede publicar el dispositivo en el catálogo de dispositivos con certificación de IoT.
- Si recibe un mensaje de notificación indicándole que falta el identificador de Microsoft Partner Network, debe [crear una cuenta del Centro de partners](https://docs.microsoft.com/partner-center/mpn-create-a-partner-center-account). Una vez aprobada su cuenta, puede volver al portal y publicar el dispositivo. Para más información, consulte [Preguntas frecuentes sobre el Centro de partners](https://support.microsoft.com/help/4340639/partner-center-account-faqs).

Antes de publicar el dispositivo en el catálogo de dispositivos con certificación de IoT, lea y acepte el contrato del programa. Puede encontrar el acuerdo en la página Configuración del portal de Azure Certified for IoT.
