# <a name="ibm-workloads-on-azure"></a>Cargas de trabajo de IBM en Azure

Muchos IBM mainframe las cargas de trabajo en función de z/OS se pueden replicar en Azure sin pérdida de funcionalidad y sin que los usuarios incluso observar los cambios en sus sistemas subyacentes. Rehospedar aplicaciones en Azure le proporciona las características de tipo mainframe que necesita además la elasticidad, disponibilidad, y ahorro de la nube.

Azure admite la integración con los entornos de gran sistema IBM existentes, lo que le permite migrar el applicates que tengan sentido, ejecutar soluciones híbridas donde sea necesario y migrar con el tiempo. Aunque completamente, puede volver a escribir programas existentes basadas en grandes sistemas de Azure, es más habitual de rehost ellos. Volver a escribir agrega costo, la complejidad y tiempo a los proyectos de migración. Con el rehospedaje, puede:

- Mover aplicaciones a un emulador en la nube.

- Migrar la base de datos a una base de datos en la nube.

- Reemplace los módulos y el código mediante motores de transformación de código.

Además, el software de IBM, incluidos WebSphere y MQ, es ahora en Azure Marketplace. Con una licencia de software de IBM, puede aprovechar las ventajas que aporta el escalado de la infraestructura a petición que ofrece Azure para poner en marcha una máquina virtual con rapidez.

Un ecosistema amplio asociado está disponible para ayudarle a migrar sistemas (mainframes) de IBM en Azure. La mayoría sigue un enfoque pragmático de reutilización siempre que sea posible antes de embarcarse en una implementación por fases de volver a escribir o reemplazar las aplicaciones. Obtenga más información y ayuda de asociados en el [centro de migración de Azure Mainframe](https://azure.microsoft.com/migration/mainframe/).
