# <a name="azure-sql-database-glossary-of-terms"></a>Glosario de términos de Azure SQL Database

|Context|Término|Más información|
|:---|:---|:---|
|Servicio de Azure|Azure SQL Database o SQL Database|[Servicio Azure SQL Database](sql-database-technical-overview.md)|
|Modelo de compra|Modelo de compra basado en DTU|[Modelo de compra basado en DTU](sql-database-service-tiers-dtu.md)|
||Modelo de compra basado en núcleo virtual|[Modelo de compra basado en núcleo virtual](sql-database-service-tiers-vcore.md)|
|Opción de implementación |Base de datos única|[Bases de datos únicas](sql-database-single-database.md)|
||Grupo elástico|[Grupo elástico](sql-database-elastic-pool.md)|
||instancia administrada|[Instancia administrada](sql-database-managed-instance.md)|
|Nivel de servicio|Básico, Estándar, Premium, De uso general, Hiperescala, Crítico para la empresa|En el caso de los niveles de servicio en el modelo núcleo virtual, vea la información sobre [base de datos única y grupo elástico](sql-database-service-tiers-vcore.md#service-tiers) e [instancia administrada](sql-database-managed-instance.md#managed-instance-service-tiers). Para los niveles de servicio en el modelo de DTU, consulte la información sobre el [modelo de DTU](sql-database-service-tiers-dtu.md#compare-the-dtu-based-service-tiers).|
|Nivel de proceso|Proceso sin servidor|[Proceso sin servidor](sql-database-service-tiers-vcore.md#compute-tiers)
||Proceso aprovisionado|[Proceso aprovisionado](sql-database-service-tiers-vcore.md#compute-tiers)

||Servidor de instancia administrada de SQL Database, servidor de instancia administrada o servidor de instancia|[Instancia administrada](sql-database-managed-instance.md)|
|Tipo de recurso|Núcleo virtual|Un núcleo de CPU proporcionado para el recurso de proceso para una única base de datos, grupo elástico o instancia administrada.|
||Tamaño de proceso y cantidad de almacenamiento|El tamaño de proceso es la cantidad máxima de CPU, memoria y otros recursos no relacionados con el almacenamiento disponibles para una única base de datos, grupo elástico o instancia administrada.  El tamaño de almacenamiento es la cantidad máxima de almacenamiento disponible para una única base de datos, grupo elástico o instancia administrada.  Para obtener más opciones de ajuste de tamaño en el modelo de núcleo virtual, vea [Bases de datos únicas de núcleo virtual](sql-database-vcore-resource-limits-single-databases.md), [Grupos elásticos de núcleo virtual](sql-database-vcore-resource-limits-elastic-pools.md) e [Instancias administradas](sql-database-managed-instance-resource-limits.md).  Para obtener más opciones de ajuste de tamaño en el modelo de DTU, vea [Bases de datos únicas de DTU](sql-database-dtu-resource-limits-single-databases.md) y [Grupos elásticos de DTU](sql-database-dtu-resource-limits-elastic-pools.md).
