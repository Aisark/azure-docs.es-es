> [!TIP]
> Azure Storage admite la autorización del acceso a los recursos de almacenamiento de blob y cola mediante Azure Active Directory (Azure AD). La autorización de usuarios o aplicaciones mediante un token de OAuth 2.0 devuelto por Azure AD proporciona una seguridad superior y facilidad de uso sobre la autorización de clave compartida y firmas de acceso compartido (SAS). Con Azure AD, no hay ninguna necesidad de almacenar la clave de acceso de cuenta con su código y arriesgarse a posibles vulnerabilidades de seguridad.
>
> Microsoft recomienda usar Azure AD con las aplicaciones de Azure Storage cuando sea posible. Para más información, consulte [Autenticación del acceso a blobs y colas de Azure con Azure Active Directory](../articles/storage/common/storage-auth-aad.md).
